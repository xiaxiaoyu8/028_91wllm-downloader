# 91wllm 批量文件获取器使用说明

## 一、运行前准备

1. 用 Chrome 或 Edge 登录 91wllm 后台。
2. 确认后台页面能正常看到就业信息列表。
3. 准备一个普通输出目录，例如桌面上的“就业证明下载”。不要选 `C:\Windows`、`Program Files` 等系统目录。

## 二、获取 Cookie

Cookie 是临时登录凭证，本工具用它访问后台列表和附件。

1. 在 Chrome 或 Edge 中打开已登录的 91wllm 后台页面。
2. 按 `F12` 打开开发者工具。
3. 点击 `Network` 或 `网络` 标签。
4. 刷新页面。
5. 在请求列表里点击加载就业信息列表的请求，地址通常包含 `/admin/tempdb/jinfo/list/`。
6. 在右侧找到 `Headers` 或 `标头`。
7. 找到 `Request Headers` 或 `请求标头`。
8. 找到 `Cookie` 这一行，复制整行内容，例如：

```text
Cookie: PHPSESSID=xxxx; Hm_lvt_xxx=xxxx; ...
```

9. 回到工具，把整行粘贴到“Cookie 粘贴区”。

注意：不要把 Cookie 发给别人，不要截图公开。Cookie 过期后，请重新登录后台并重新复制。

## 三、第一次测试下载

1. 双击 `91wllm批量文件获取器.exe`。
2. 粘贴 Cookie。
3. 选择输出目录。
4. 下载范围选择“仅测试前 5 个”。
5. 点击“开始下载”。
6. 下载完成后点击“打开目录”，检查文件名和班级文件夹是否正确。

## 四、正式批量下载

1. 确认测试文件正常。
2. 下载范围改为“全部下载”。
3. 点击“开始下载”。
4. 等待完成。

文件会保存为：

```text
输出目录/班级/学号_姓名_序号.ext
```

日志文件会保存在输出目录根目录：

- `download_log.csv`：成功下载或已存在跳过的记录。
- `metadata_failures.csv`：班级、学号、姓名识别不完整的记录。
- `failed_downloads.csv`：下载失败的记录。

## 五、高级设置说明

看不懂高级设置时可以不改。

- `START_URL`：开始地址，默认是当前就业后台列表地址。
- `OUT_DIR`：输出目录，由首页“输出目录”选择。
- `LIMIT`：下载范围。“仅测试前 N 个”对应 N；“全部下载”对应无限制。
- `INSECURE`：禁用 TLS 证书校验。当前站点如证书异常，建议保持勾选。
- `HEADFUL`：显示浏览器窗口。排查问题时可以勾选。
- `GRID_SELECTOR`：表格行选择器，默认 `#jinfo-grid table tbody tr`。
- `NEXT_SELECTOR`：下一页按钮选择器。留空时程序会自动尝试识别“下一页”。
- `NO_PAGINATION`：只处理当前页。
- `MAX_PAGES`：最多处理多少页，防止异常无限翻页。
- `TIMEOUT_MS`：页面等待超时，单位毫秒。
- `CLASS_COL`、`STUDENT_ID_COL`、`NAME_COL`、`ATTACHMENT_COL`：班级、学号、姓名、附件列号。从 1 开始，留空表示自动识别。

## 六、常见问题

### 提示 Cookie 过期或登录页

重新登录后台，按上面步骤重新复制 Cookie，再粘贴运行。

### 提示输出目录没有写入权限

优先换到桌面或文档里的普通文件夹。如果必须写入受保护目录，请右键 exe 选择“以管理员身份运行”。

### 下载数量不对

先确认后台筛选条件是否正确，再检查高级设置里的开始地址和分页设置。

### 文件已经存在

工具会跳过同名已存在文件，并在 `download_log.csv` 中记录“已存在，跳过”。

## 七、打包说明

开发者在项目目录中运行：

```powershell
.\build_exe.ps1
```

打包完成后，把 `dist/91wllm批量文件获取器` 整个文件夹发给用户。不要只发送 exe，因为文件夹里还包含浏览器运行资源和说明文件。
